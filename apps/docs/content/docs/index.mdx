# open-wa / wa-automate

`open-wa` gives you a few different ways to automate WhatsApp, and the docs are now organized around those real usage modes instead of the old autogenerated Docusaurus tree.

## Start here

- Use **[Getting Started](/docs/getting-started/easy-api)** if you want a running Easy API quickly.
- Use **[Custom Code](/docs/getting-started/custom-code)** if you are embedding the library in your own Node.js project.
- Use **[Socket Client](/docs/client-and-integrations/socket-client)** if you want to connect to a remote Easy API instance.
- Use **[Cloudflare Session Proxy](/docs/client-and-integrations/cf-proxy)** if you want remote session access without exposing a public port.

## Documentation map

- **Getting Started** — first-run guides for Easy API, Docker, custom code, and link-code login.
- **Guides** — practical guides for config, events, sessions, messages, media, and groups.
- **Client & Integrations** — SocketClient, Chatwoot, proxying, and Cloudflare session proxy.
- **Concepts** — the runtime model and core terminology.
- **API Reference** — generated API docs from the active reference set.
- **Operations & Troubleshooting** — error handling, logout recovery, and production best practices.
- **Licensing** — licensed features and feature-availability notes.

## Migration notes

This Fumadocs site intentionally does **not** mirror the old legacy sidebar one-to-one. The legacy site had ordering collisions, flat how-to sprawl, and orphaned docs outside the main tree. Content has been regrouped so task-oriented material is easier to find and maintain.
