# open-wa / wa-automate

`open-wa` gives you a few different ways to automate WhatsApp, and the docs are now organized around those real usage modes instead of the old autogenerated Docusaurus tree.

## What you can build

- Customer support inboxes that sync WhatsApp into your own tools
- Order, booking, and delivery notifications from internal systems
- Bots that react to messages, group activity, and runtime events
- Webhook bridges for CRMs, helpdesks, automations, and low-code tools
- Multi-session automations with named accounts and isolated consumers
- AI-agent workflows through the built-in Model Context Protocol server

## Prerequisites

- **Node.js** >= 22.21.1
- **npm** or **pnpm** (pnpm >= 10.25.0 for repo development)
- A WhatsApp account on a phone that can stay connected

## v5 Alpha Status

<Callout type="warning">
v5 is currently in alpha. Use v4 (4.76.0) for production systems unless you are intentionally testing v5.
</Callout>

The current stable line is **v4.76.0**. v5 is a rewrite with a cleaner architecture but may have issues:

```bash
# Production: use v4
npx @open-wa/wa-automate@4.76.0

# Testing: use v5 alpha
npx @open-wa/wa-automate
```

### What changed in v5

- Monorepo architecture with separate packages
- Pluggable browser drivers (Puppeteer, Playwright, Lightpanda)
- `createClient` as the new public contract
- Plugin system with SDK for reusable integrations
- Built-in MCP server for AI agent integration

### When to use v5 vs v4

- **v4**: Production systems, mature deployments, existing integrations
- **v5**: Testing new features, contributing to development, fresh projects willing to track alpha changes

## Start here

- Use **[Getting Started](/docs/getting-started/easy-api)** if you want a running Easy API quickly.
- Use **[Custom Code](/docs/getting-started/custom-code)** if you are embedding the library in your own Node.js project.
- Use **[Socket Client](/docs/client-and-integrations/socket-client)** if you want to connect to a remote Easy API instance.
- Use **[Cloudflare Session Proxy](/docs/client-and-integrations/cf-proxy)** if you want remote session access without exposing a public port.
