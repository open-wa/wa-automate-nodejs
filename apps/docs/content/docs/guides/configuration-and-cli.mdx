# Configuration and CLI

The old legacy docs split configuration across several small pages. This page brings the essentials together.

## Common config fields

Use the config object passed to `create()` for library/runtime behavior such as:

- `sessionId` ‚Äî stable session naming
- `headless` ‚Äî visible vs background browser
- `useChrome` / `executablePath` ‚Äî browser selection
- `qrTimeout` / `authTimeout` ‚Äî authentication timing
- `maxQr` ‚Äî limit QR emission count
- `cacheEnabled` ‚Äî page/runtime memory trade-off
- `licenseKey` ‚Äî feature unlocks where applicable
- `proxyServerCredentials` ‚Äî outbound proxy support
- `linkCode` ‚Äî link-code login flow

## Example

```ts
const client = await create({
  sessionId: 'sales',
  useChrome: true,
  headless: true,
  qrTimeout: 0,
  authTimeout: 60,
});
```

## High-value CLI flags

```bash
# networking
--port 8080
--api-key "your-secure-key"
--webhook "https://your-app.example/webhooks/open-wa"
--api-host "https://api.example.com"

# session identity
--session-id sales
--link-code "447123456789"

# runtime helpers
--pm2
--tunnel
--multi-device

# integrations
--chatwoot-url "https://app.chatwoot.com/api/v1/accounts/123"
--chatwoot-api-access-token "token"

# licensing
--license-key "YOUR-LICENSE-KEY"
```
