# Pricing and licensing

This page is the decision surface for open-wa licensing. If you already know you
need a key and only want the runtime details, jump to
**[Licensed features](/docs/licensing/licensed-features)**.

<Callout type="note" title="What this page is for">
  Use this page when you are trying to decide whether the free/community surface
  is enough, whether you need a restricted license for gated flows, or whether
  you need insiders access for advanced or early-access capabilities.
</Callout>

## What changes when you upgrade

<Steps>
  <Step title="Restricted licensing unlocks gated production use cases">
    This is the path when the community surface is not enough and you need a key
    for host-account-specific or gated runtime behavior.
  </Step>
  <Step title="Insiders licensing is for advanced or early-access capability">
    Choose this when the restricted tier is still too narrow and your workflows
    depend on higher-end or earlier-surface features.
  </Step>
  <Step title="The session context still matters">
    A license decision is not just about owning a key. The real runtime session,
    host account, and deployment context still determine whether the behavior you
    care about is available where you need it.
  </Step>
</Steps>
