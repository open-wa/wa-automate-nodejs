## Expanded usage notes

### Request examples

Archive and unarchive a chat:

```ts
await client.chats.archive('447123456789@c.us');
await client.chats.unarchive('447123456789@c.us');
```

Mute a chat for eight hours, then unmute it:

```ts
const eightHoursMs = 8 * 60 * 60 * 1000;
await client.chats.mute('447123456789@c.us', eightHoursMs);
await client.chats.unmute('447123456789@c.us');
```

Pin a chat, then remove the pin:

```ts
await client.chats.pin('447123456789@c.us');
await client.chats.unpin('447123456789@c.us');
```

Set disappearing messages. Common values are `false` to disable, or a duration in seconds when WhatsApp accepts it:

```ts
await client.chats.setEphemeral('447123456789@c.us', 86400);
await client.chats.setEphemeral('447123456789@c.us', false);
```

Clear and delete are destructive. Use them only after your own confirmation step:

```ts
await client.chats.clear('447123456789@c.us');
await client.chats.delete('447123456789@c.us');
```

### Response examples

Most chat update methods return a boolean when WhatsApp accepts the change:

```json
true
```

`getChat` returns a chat object from WhatsApp Web:

```json
{
  "id": "447123456789@c.us",
  "name": "Example Contact",
  "isGroup": false,
  "unreadCount": 0,
  "archive": false,
  "pin": 0,
  "muteExpiration": 0
}
```

### Error cases

- The chat ID is malformed or points to a chat not loaded in the session.
- WhatsApp Web is offline or the session is not ready.
- Pinning fails because WhatsApp enforces a maximum number of pinned chats.
- Mute or ephemeral values are not accepted by the current WhatsApp Web build.
- A destructive action fails because the chat store changed during the request.

### Permission and license requirements

The listed chat methods do not require a special license. They do require an authenticated session, and Easy API calls require the configured API key when one is enabled.

### Caveats and limitations

Archive, mute, pin, and ephemeral settings change the host account state in WhatsApp. They are not local API preferences. `clearChat` removes messages from the host account's visible chat history. `deleteChat` removes the chat thread from the host account. Neither method should be used as a reversible storage operation.
