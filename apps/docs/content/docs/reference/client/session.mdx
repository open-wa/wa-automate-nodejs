## Expanded usage notes

### Health check and readiness

Use `healthCheck`, `getConnectionState`, and `getMe` together when deciding whether your app can send messages. A process can be alive while WhatsApp Web is still syncing, so a readiness check should confirm both API reachability and session state.

### Request examples

Basic readiness check:

```ts
const health = await client.session.healthCheck();
const state = await client.session.getConnectionState();
const me = await client.session.getMe();
```

License and runtime status:

```ts
const licenseType = await client.session.getLicenseType();
const features = await client.session.getFeatures();
const version = await client.session.getWAVersion();
```

Take a snapshot of the current WhatsApp Web page:

```ts
const screenshot = await client.session.getSnapshot(undefined, 1280, 720);
```

Easy API examples:

```bash
curl "http://localhost:8080/api/session/healthCheck" \
  -H "X-API-Key: your-secure-key"

curl "http://localhost:8080/api/session/getLicenseType" \
  -H "X-API-Key: your-secure-key"
```

### Response examples

Health check output can include process and session readiness fields:

```json
{
  "ok": true,
  "sessionId": "sales",
  "state": "CONNECTED",
  "ready": true,
  "uptime": 3600
}
```

License status can be a string, object, or `false` depending on runtime state:

```json
{
  "type": "insiders",
  "valid": true,
  "features": ["insiders", "restricted"]
}
```

Snapshot output is a string, usually a data URL:

```json
"data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA..."
```

A host account response may look like this:

```json
{
  "id": "441234567890@c.us",
  "pushname": "OpenWA Bot",
  "me": "441234567890@c.us"
}
```

### Error cases

- The process is running but the WhatsApp session is not authenticated yet.
- The browser page crashed or was closed.
- Snapshot fails because the page is not available or the requested dimensions are invalid.
- License data cannot be verified because no key is configured or the license server is unreachable.
- Personal-only methods such as `setMyName` are called on a business account where WhatsApp blocks the change.
- Insiders methods such as `setProfilePicture` are called without an Insiders license.

### Permission and license requirements

Most session read methods do not require a special license. `setProfilePicture` requires an Insiders license. `setMyName` is personal-only. Easy API calls require the configured API key when one is enabled.

### Caveats and limitations

Battery and plugged status may be unavailable or stale on multi-device sessions. `getLoadedMessageCount` counts messages loaded in the current web session, not all account history. Snapshot data can be large, so avoid polling it. Treat health check output as a readiness signal, not as a permanent guarantee that the next WhatsApp action will succeed.
