## Expanded usage notes

### Chat ID formats

`getAllChatIds` returns the chat identifiers currently known to the session. The most common formats are:

```text
447123456789@c.us              personal or business contact chat
120363025555555555@g.us        group chat
447123456789-1445627445@g.us   older group chat format
123456789012345@lid            linked identity chat ID
status@broadcast               WhatsApp status broadcast thread
```

`@c.us` IDs represent contact chats. `@g.us` IDs represent groups and communities. `@lid` IDs are linked identity IDs used by newer WhatsApp account identity flows. If your app stores contact IDs, keep the original ID returned by the API and normalize only at the boundary where a method requires a specific format.

### Normalization example

Normalize phone numbers before calling methods that require a contact JID:

```ts
function toContactId(input: string) {
  const digits = input.replace(/\D/g, '');
  return `${digits}@c.us`;
}

const id = toContactId('+44 7123 456789');
await client.messages.sendText(id, 'Hello');
```

Do not convert `@lid` to `@c.us` unless you have a trusted mapping from the same session. LID values can be the only stable identifier for some chats.

### Error cases

- The session is not ready, so the chat store is empty or still loading.
- A consumer assumes every ID is `@c.us` and rejects groups or LID chats.
- A stored number is missing a country code before normalization.
- A later method rejects `status@broadcast` because it expects a contact or group chat.

### Caveats and limitations

The returned list is a snapshot of chats loaded in WhatsApp Web. It may not include archived, very old, or not yet synced chats until WhatsApp loads them. Treat IDs as opaque strings in storage, and validate format only when a method requires a narrower contact, group, or LID type.
