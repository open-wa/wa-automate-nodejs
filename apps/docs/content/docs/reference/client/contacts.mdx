## Expanded usage notes

### Request examples

Check whether a number is registered on WhatsApp:

```ts
const status = await client.contacts.checkNumberStatus('447123456789@c.us');
```

Get a profile picture URL:

```ts
const pictureUrl = await client.contacts.getProfilePicture('447123456789@c.us');
```

Block and unblock a contact:

```ts
await client.contacts.block('447123456789@c.us');
await client.contacts.unblock('447123456789@c.us');
```

List blocked contact IDs:

```ts
const blocked = await client.contacts.getBlockedIds();
```

### Response examples

Number status responses are controlled by WhatsApp and may include registration and JID data:

```json
{
  "id": "447123456789@c.us",
  "status": 200,
  "numberExists": true,
  "canReceiveMessage": true
}
```

A profile picture call returns a URL string when available:

```json
"https://pps.whatsapp.net/v/t61.24694-24/example-image"
```

Blocked IDs are returned as contact IDs:

```json
[
  "447123456789@c.us",
  "441234567890@c.us"
]
```

### Error cases

- The contact ID is not normalized as `number@c.us`.
- `checkNumberStatus` or `getNumberProfile` is called without an Insiders license.
- The contact has no visible profile picture or privacy settings block it.
- WhatsApp rate limits number checks after repeated lookups.
- Block or unblock fails because the contact is not in a state WhatsApp accepts.

### Permission and license requirements

`checkNumberStatus` and `getNumberProfile` require an Insiders license. Profile picture, read receipt, common groups, block, unblock, and blocked ID methods do not require a special license. Easy API calls still require the configured API key.

### Caveats and limitations

Profile picture URLs are temporary and should not be stored as permanent assets. Blocked IDs reflect the host account, not a global block list. Number status checks should be used sparingly because WhatsApp may throttle or hide results based on account trust and recent activity.
