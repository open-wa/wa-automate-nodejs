Optional executablePath

executablePath: string
  • Defined in api/model/index.ts:167

Some features, like video upload, do not work without a chrome instance. Puppeteer only provides a chromium instance out of the box. Set this to the path of your chrome instance or you can use useChrome:true to automatically detect a chrome instance for you.

  • Defined in api/model/index.ts:151

By default, all instances of @open-wa/wa-automate are headless (i.e you don't see a chrome window open), you can set this to false to show the chrome/chromium window.

Optional restartOnCrash

restartOnCrash: (value: Whatsapp) => any | Function
  • Defined in api/model/index.ts:184

If set, the program will try to recreate itself when the page crashes. You have to pass the function that you want called upon restart. Please note that when the page crashes you may miss some messages. E.g:

const start  = async (client: Whatsapp) => {...}
create({
...
restartOnCrash: start,
...
})

Optional sessionData

sessionData: SessionData
  • Defined in api/model/index.ts:92

JSON object that is required to migrate a session from one instance to another or ot just restart an existing instance. This sessionData is provided in a generated JSON file upon QR scan or an event. You can capture the event like so:

import {create, ev} from '@open-wa/wa-automate';
ev.on('sessionData.**', async (sessionData, sessionId) =>{
         console.log(sessionId, sessionData)
     })

Optional sessionId
