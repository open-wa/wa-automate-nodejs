# Unfuckupgaps Learnings

- 2026-04-04: Added a single executable parity matrix at `packages/core/test/fixtures/release-blocker-parity-matrix.json` so release blockers and accepted non-blockers are traced from audit reference -> legacy file -> current file -> proof spec.
- 2026-04-04: The new red-state core probes confirm current v5 is still missing a non-launch browser->node event bridge, frame-navigation reinjection, post-auth ripe-session gating, `userDataDir` threading through `Transport.initialize()`, and a broken-method integrity gate.
- 2026-04-04: `packages/wa-automate/src/server/__tests__/readiness-parity.test.ts` stays green, which is useful evidence that host-vs-session readiness separation already exists and should remain outside the red blocker set.
- 2026-04-04: `@open-wa/driver-interface` now carries lifecycle-safe `DisposableHandle` ownership, typed `IPage.on/off` listeners, generation-safe `addInitScript()` disposal, and thin `IFrame`/`IRequest`/`IConsoleMessage` abstractions so later reinjection and interception work no longer has to reach for raw Puppeteer types.
- 2026-04-04: Adapter listener cleanup is deterministic only if each event stores exact wrapped callbacks per original handler (not a single weak entry); using per-handler wrapper sets lets `dispose()` remove one registration while `off()` can still clear the remaining wrappers safely.
- 2026-04-04: Playwright request interception stays parity-safe when `setRequestInterception()` reuses one stable route handler and `IRequest.abort/continue/respond` throw an explicit bound-route error instead of pretending request-event payloads can mutate traffic on their own.
- 2026-04-04: The active v5 boot path now threads `userDataDir` from config/CLI/createClient into `Transport.initialize()` and the driver launch call, while Transport only enables request interception for the parity-critical subset (`blockCrashLogs`, headless `blockAssets`) and logs an explicit downgrade warning if interception cannot be activated.
- 2026-04-04: `packages/config` JSON-schema generation is stable under the workspace `tsx` runtime; using `ts-node` against the schema source hit Node 24 strip-only TypeScript limitations around enums and ESM resolution.
- 2026-04-04: The v2 controller shell now owns persistent boot infrastructure in `packages/core/src/transport/InjectionController.ts`; `ProgressBarEvent`, `CriticalInternalMessage`, and the `prog_observer` init script are registered during `Transport.initialize()` so they exist before the first `goto()` navigation window.
- 2026-04-04: Moving init-script registration earlier exposed that `ScriptLoader` must resolve both source-time `src/transport/assets` and bundled `dist/transport/assets` layouts; candidate-path resolution keeps pre-navigation asset loading working in tests and builds.
- 2026-04-04: The real runtime event bridge is safest when `InjectionController` owns both persistent exposed functions and per-document runtime bridge state; storing a page-side `__OPENWA_RUNTIME_BRIDGE__` keyed by generated runtime id makes repeat setup a no-op for the same generation while still allowing fresh WAPI listener registration after navigation.
- 2026-04-04: Page-side bridge ownership is safer when `__OPENWA_RUNTIME_BRIDGE__` stores active listener entries plus optional disposer handles and the current `window.WAPI` object reference; that lets `InjectionController` treat same-generation setup as a no-op, but still reset or dispose listeners when the runtime object is replaced or the controller rolls the runtime generation.
- 2026-04-04: Navigation/runtime recovery is safest when `Transport` collapses reinjection requests behind a latest-request id and waits for `InjectionController.waitForIdle()` before probing; that prevents stale main-frame navigations from rebinding bridges after a newer generation has already taken over.
- 2026-04-04: Post-auth parity is safest when fresh-auth and resumed-session both pass through one transport-level reconciliation path that reuses the runtime-recovery helper, but only fresh-auth forces an immediate reinjection attempt; a separate `waitForRipeSession()` gate before patch/license overlays keeps later finalization from running against a half-settled authenticated runtime.
- 2026-04-04: Loaded-equivalent behavior needs an explicit awaited hook in `createClient()` instead of an async event listener, because `HyperEmitter.emit()` does not await listener promises; registering facade finalization hooks let `Client.loaded()` finish sync waiting, listener autobinding, and phone-version capture before `client.ready` emits.
- 2026-04-04: Client listener autobinding is safer as a one-time hub (`ListenerManager.autobindAll()`) than per-callback direct subscriptions; it preserves the loaded-phase side effect without forcing user listeners to exist before runtime events start flowing.
- 2026-04-04: Logout parity is safest when the retained `session.logout` hook stays explicit in `packages/client/src/Client.ts` and waits for `ListenerManager.waitForQueuesToDrain()` before invalidating session data, optionally deleting persisted state, and optionally stopping the client; threading the two logout flags plus `sessionDataPath` through `@open-wa/core` keeps that ordering testable without hiding it in unrelated lifecycle code.
- 2026-04-04: The broken-method parity gap closes cleanly when transport validation asks `InjectionController` for the required runtime WAPI method inventory, treats missing required bridge methods as an explicit `required_method_missing` blocker, and routes repair through the existing runtime recovery/reinjection helper instead of adding a second ad-hoc integrity path.
- 2026-04-04: Helper-asset truth for shipped `packages/core/src/transport/assets/wapi.js` is now explicit and enforced in `ScriptLoader`: core ships only `init_patch.js`, `prog_observer.js`, `wapi.js`, and `launch.js`, and loader audit now fails if `wapi.js` reintroduces legacy helper globals like `axios`, `jsSHA`, `window.Base64`, `QRCode`, `CryptoJS`, or `objectHash`.
- 2026-04-04: Internal-event-handler truth in core is now honest: the deferred init patch no longer calls a fake `injectInternalEventHandler()` stub, because the real browser->node runtime contract already lives in `InjectionController.registerRuntimeWapiBridge(...)` and its transport bindings/tests.
- 2026-04-04: Patch lifecycle parity stays least ambiguous when `cachedPatch` is truly opt-in, `ghPatch` explicitly means “GitHub raw is the primary patch source,” cached hits trigger a best-effort background refresh, and the legacy `init_patch.js` remains a required deferred patch applied only after license handling so any failure blocks `patchLifecycle` readiness instead of being hidden inside preload semantics.
- 2026-04-04: License readiness stays truthful when server-confirmed unlocks and `local_metadata` fallbacks use different statuses (`valid` vs `metadata_only`); metadata-only can remain explicitly non-blocking for bootstrap, but it must not satisfy `licenseLifecycle` or look equivalent to the legacy server-payload path.
- 2026-04-04: Remaining auth parity is now explicit in core bootstrap instead of collapsing into generic timeout: `waitForAuthentication()` classifies invalid-session/NUKE, phone-out-of-reach, qrMax exhaustion, and link-code generation separately, while `createClient()` maps each retained path to an honest fatal scope/message and focused bootstrap-contract tests prove the distinctions.
- 2026-04-04: Readiness truth stays honest when `createClient()` derives `getReadiness()` from live transport operational state instead of only stored requirement flags; gating `READY`/`client.ready` on active generation, operational runtime+bridge truth, and a settled runtime-recovery queue prevents stale navigation/reinjection work from satisfying session readiness.
- 2026-04-04: Facade/runtime truth is safest when `packages/client` keeps an executable inventory of every declared method/listener, marks unsupported entries explicitly, and smoke-tests that inventory against the terminal `window.WAPI.*` assignments in shipped `wapi.js` plus the core transport listener surface exported from `packages/core/src/transport/runtimeListenerSurface.ts`.
- 2026-04-04: Public contract truth is least confusing when `@open-wa/wa-automate` re-exports `Config` and `ConfigSchema` from `@open-wa/config` directly, while config field descriptions and smoke tests explicitly mark popup QR as downgraded and `.data.json` session restore as deprecated in favor of `userDataDir`.
- 2026-04-04: The final RC evidence pack is safest when one manifest is machine-validated against `packages/core/test/fixtures/release-blocker-parity-matrix.json`; that keeps accepted non-blockers (`popup_qr_parity`, `json_session_restore`) visibly separate from the seven blocker proofs instead of relying on a handwritten checklist.
- 2026-04-04: `PluginHost` runtime already matched the new plugin SDK contract; the red blocker was a stale unit suite still calling the removed `register(name, plugin, input)` shape. Rebuilding those tests around real plugins with `meta.name` and current register options cleared the blocker without changing host behavior.
- 2026-04-04: Core release verification is more deterministic when the live-browser QR E2E is opt-in via `OPENWA_CORE_E2E=true` instead of always running under `pnpm --filter @open-wa/core test`; that preserves the smoke path for explicit environments while keeping the default suite stable.
- 2026-04-04: `packages/wa-automate` needed the same Node-facing compiler mode as other runtime packages (`module`/`moduleResolution: NodeNext`, `target/lib: ES2022`) so TypeScript could resolve package-export subpaths like `@open-wa/schema/methods` and workspace packages like `@open-wa/driver-puppeteer`; the one remaining blocker was a test-only `import.meta` usage inside this CommonJS package, which was safest to replace with the existing `__dirname` path primitive.
- 2026-04-04: Backfilled plan-required task-2 through task-19 evidence files under `.sisyphus/evidence/` from the existing green task-20 manifest/release-suite plus the dated `unfuckupgaps` learnings/issues trail, without changing implementation code.
- 2026-04-04: The QR bootstrap blocker in `createClient()` closes cleanly by deferring only pre-auth `post_injection` `required_method_missing` results when `sessionLoaded === false`, keeping `runtimeUsable` pending/truthful until auth settles, then rerunning the same strict `post_injection` gate after `reconcilePostAuthRuntime()` before overlays continue.
