# Facility Routes Refactoring: Path Variables & Aggregates

Currently, the operator tooling in the `apps/webapp` relies on a `?facilityId=...` search parameter constraint. By moving from **search-parameter scoping** mapped on flat routes to **path-parameter directories** mapped with an `index` route, we will easily support parameterized detail views alongside general aggregate views.

## User Review Required

> [!WARNING]
> Moving to an `index` (aggregate) view requires that the backend API can serve aggregated data across *all* owned/managed facilities. I will update the backend endpoints to not strictly enforce a single `facilityId`, or if they currently do, build aggregator endpoints.

## Proposed Routing Changes

Each file will be moved to a path-parameter scope: `$facilityId.tsx`. In TanStack router, this automatically supplies the path parameter string. Then, new `index.tsx` files will be structured to handle the aggregate root requests.

### Route Rewrites & TanStack Pitfalls to address during Move:
1. **`createFileRoute` ID updates**: When moving `facility-financials.tsx` to `facility-financials/$facilityId.tsx`, the `createFileRoute("/_protected/facility-financials")` must be updated to `createFileRoute("/_protected/facility-financials/$facilityId")`.
2. **Search Schema Validation**: The Zod schemas currently enforce `facilityId: z.string()` via `validateSearch`. We MUST remove `facilityId` from these schemas entirely for the `$facilityId.tsx` variants.
3. **Param Extraction**: Any usage of `Route.useSearch().facilityId` inside the components MUST be replaced with `Route.useParams().facilityId`.

## ⚠️ Hardcoded URL String Pitfalls ⚠️
During my final double-check, I discovered multiple backend files and utility functions that bypass Tanstack's `<Link>` and manually concatenate `?facilityId=`. These **must** be refactored:

1. **Backend Notifications (`apps/api/src/routes/campaigns.ts`)**:
   - `ctaHref: /facility-campaigns?facilityId=${data.facilityId}` -> `ctaHref: /facility-campaigns/${data.facilityId}`
2. **Webapp Builders (`apps/webapp/src/routes/_protected/facility-settings.tsx` & `facility.schedule.tsx`)**:
   - `buildFacilitySettingsHref`: `/facility-settings?facilityId=...` -> `/facility-settings/...`
   - `buildFacilityScheduleHref`: `/facility/schedule?facilityId=...` -> `/facility-schedule/...`
   - `buildManageCourtsHref`: `/manage-courts?facilityId=...` -> `/manage-courts/...`
3. **Webapp Quick Links (`apps/webapp/src/routes/_protected/account/$accountView.tsx`)**:
   - `FACILITY_QUICK_LINKS` will map uniformly to the `index` endpoints.
4. **Site-wide Declarative Links**:
   - Updates from `to="/facility-financials" search={{ facilityId }}` to `to="/facility-financials/$facilityId" params={{ facilityId }}`.
